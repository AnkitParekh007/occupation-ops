# When To Use Occupation-Ops Vs Job Search Pipeline Tools

Occupation-Ops is designed for the stage before active application operations.

## Use Occupation-Ops when

- your GitHub and portfolio do not yet prove the role clearly
- your profile sounds generic or vague
- your proof projects lack screenshots, recruiter takeaways, or README clarity
- you need interview-safe stories from real work
- you need a weekly proof-shipping plan more than another tracker

## Use job-search pipeline tools when

- your proof surfaces are already strong
- you are actively scanning portals and triaging openings
- you need application tracking and workflow automation
- you are optimizing outreach volume or follow-up operations

## Competitive position

Occupation-Ops should win on:

- clearer niche ownership
- stronger proof scoring
- better recruiter-scan outputs
- more honest public-proof guardrails
- tighter flagship workflow for AI/frontend pivots

It should not try to win by becoming a smaller clone of a tracker/scanner tool.
