# Ethics

Occupation-Ops is a career-tech workflow, not a shortcut around truth.

## Core ethical rules

- do not fake experience
- do not fake production usage
- do not fake merged pull requests
- do not fake endorsements, testimonials, or star counts
- do not mass-apply or spam recruiters
- do not automate third-party sites against their terms

## What the repo should optimize for

- honest positioning
- stronger public proof
- better README quality
- more credible GitHub profiles
- interview-safe storytelling
- steady weekly execution

## What the repo should not optimize for

- volume over credibility
- hype over evidence
- ATS folklore
- deceptive resume wording
- synthetic hiring success stories

## Good use cases

- clarify a target role
- identify missing proof projects
- improve a GitHub profile README
- plan a 30-day portfolio sprint
- turn real work into interview stories

## Bad use cases

- generating fabricated achievement bullets
- rewriting a profile to imply experience the user does not have
- creating fake public proof without real work behind it
- automating recruiter outreach at scale
