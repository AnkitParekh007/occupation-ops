# Recruiter Review Guide

This guide explains how a recruiter or hiring manager can read an Occupation-Ops profile output realistically.

## What Occupation-Ops can help reveal

- how clearly a candidate understands their target role
- whether their GitHub proof matches their stated direction
- whether their projects are explained in recruiter-readable language
- whether they can distinguish shipped work, mock systems, and planned work
- whether they are improving credibility over time

## What Occupation-Ops does not prove

- guaranteed job readiness
- guaranteed interview performance
- guaranteed ATS success
- guaranteed production experience
- guaranteed employment outcomes

## Best recruiter signals to look for

- role-specific GitHub profile headline
- README sections such as "what this proves" or "mocked vs real"
- proof projects with screenshots, run instructions, and architecture notes
- interview stories tied to real public repos
- evidence of consistent weekly shipping or iteration

## How to evaluate candidate outputs fairly

1. Look for clarity, not inflation.
2. Check whether claims are tied to visible artifacts.
3. Distinguish planned work from shipped work.
4. Prefer steady proof-building over exaggerated positioning.
