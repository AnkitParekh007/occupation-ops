# FAQ

## Is this a resume faker?

No. The product direction is explicitly proof-first and truthfulness-first. It should highlight gaps when proof is weak rather than invent confidence.

## Is this a job bot or mass-apply tool?

No. Occupation-Ops is not a tracker, scanner, portal scraper, recruiter CRM, or auto-apply workflow.

## Do I need an API key?

No. The current repo is local-first and file-based.

## What do I actually get?

The flagship flow produces:

- Markdown
- JSON
- HTML

The HTML output is the easiest format for public demos and quick review.

## Who is this for?

The current product wedge is AI/frontend candidates who need stronger public proof before they apply more broadly.

## What if my proof is weak today?

That is a valid input state. The tool should identify missing signals and create a backlog rather than pretending the evidence is already strong.
