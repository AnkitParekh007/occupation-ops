# Introduction

Occupation-Ops is a local-first proof engine for developers who need stronger public evidence before they apply for AI/frontend roles.

It focuses on one narrow problem:

- weak GitHub positioning
- unclear portfolio proof
- thin public artifacts
- interview stories that are not backed by visible work

It does **not** try to be a mass-apply tool, tracker, portal scanner, or ATS gaming system.

## The product wedge

The flagship workflow is the **AI Frontend Architect** dossier. It turns `profile.yml` into a role-specific output bundle:

- proof inventory
- role-gap scorecard
- public-proof metrics
- GitHub rewrite guidance
- README improvement checklist
- portfolio project briefs
- interview story map
- weekly shipping plan

## Why this exists

Many candidates do not lose because they applied too little. They lose because their proof is not strong enough for a recruiter, hiring manager, or interviewer to trust quickly.

Occupation-Ops exists to improve the quality of that proof before more outreach happens.

## Core principles

- **Proof-first**: strengthen evidence before increasing application volume.
- **Local-first**: the repo runs from your terminal and writes files locally.
- **Truthful by design**: outputs should never encourage fake metrics or invented experience.
- **Structured outputs**: the core workflows emit Markdown, JSON, and HTML.
