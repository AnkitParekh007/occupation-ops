# Scoring and Validators

Occupation-Ops should score evidence, not just describe it.

## Target proof metrics

The current direction includes metrics such as:

- proof density
- claim support ratio
- portfolio clarity
- recruiter scan readiness

## Validators

The validation layer should check the artifacts this repo claims to improve:

- GitHub profile README quality
- proof project completeness
- screenshot or demo presence
- recruiter readability of project descriptions
- truthfulness and claim-risk boundaries

## Guardrail principle

If the profile lacks proof, the output should mark a gap instead of inventing confidence.

That is a product requirement, not a copy preference.
