# Positioning

Occupation-Ops is intentionally positioned as a **proof-before-applying** system.

## Competitive stance

The goal is not feature parity with job-search pipeline tools.

The goal is category ownership for:

- public proof readiness
- recruiter-readable GitHub and portfolio positioning
- truthful evidence scoring
- pre-application credibility building

## When to use Occupation-Ops

Use it when a candidate needs:

- stronger public evidence
- a clearer proof backlog
- sharper GitHub messaging
- better interview stories tied to real work

## When not to use it

Do not use it as:

- a mass-apply engine
- a recruiter CRM
- a portal scanner
- a fake-resume generator

For a deeper comparison, see the repository doc at `docs/positioning-vs-job-search-tools.md`.
