# Frontend Engineer Track

## Goal

Show recruiter-readable proof that you can build usable, accessible,
maintainable product interfaces with clear engineering judgment.

## Proof Signals

- Portfolio link with visible screenshots or demos.
- Pinned repos that explain what each project proves.
- README files with setup, architecture, and accessibility notes.
- Examples of state management, validation, and API integration.
- Truthful boundaries between shipped work, mock work, and roadmap work.

## Strong Proof Projects

- Dashboard with real state flows and role-based UI.
- Design system or component gallery with usage guidance.
- Form-heavy workflow with validation, accessibility, and error handling.

## Interview Topics

- Component boundaries and ownership.
- State management tradeoffs.
- Accessibility decisions.
- Performance bottlenecks and fixes.
- API integration and error states.
